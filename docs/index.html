The Problem 痛点

AI mimics your worst habits AI 会模仿你最差的代码习惯

When AI reads your legacy code, it thinks that's the standard. It copies every anti-pattern it finds. 当 AI 读到旧代码时，它把那当成标准，照搬每一个反模式。

💣

God Classes keep growing God Class 越来越大

800-line file? AI adds 50 more lines instead of refactoring into smaller modules. 800 行的大文件？AI 继续往里堆 50 行，而不是拆分成小模块。

💥

any spreads like a virus any 像病毒一样扩散

One any in your codebase, AI uses it everywhere. Type safety goes out the window. 代码里有一个 any，AI 就到处用。类型安全形同虚设。

🔨

Callback hell multiplies 回调地狱层层加码

5-level nesting? AI nests 6. It treats your worst code as the template. 嵌套 5 层？AI 嵌 6 层。它把你最差的代码当模板。

⚠️

Outdated patterns persist 过时写法被延续

Using ObservableObject in Swift 6? Old enum in TypeScript? AI follows suit. Swift 6 还在用 ObservableObject？TypeScript 还在用 enum？AI 照抄不误。

The Difference 区别

Vague vs. actionable 空话 vs. 可执行

Most coding rules are too vague for AI to execute. Ours are concrete directives with code examples. 大多数编码规则太模糊，AI 无法执行。我们的每条规则都是具体指令，附带代码示例。

✅ claude-rules

  • Functions ≤30 lines, files ≤300 lines函数 ≤30 行，文件 ≤300 行
  • ref for primitives, reactive for objects基本类型用 ref，对象用 reactive
  • No any — use unknown then narrow禁止 any，用 unknown 后收窄
  • No barrel exports (causes circular deps)禁止桶导出（导致循环依赖）
  • as const + union type, not enumas const + 联合类型，不用 enum
